Rethinking Shrinkage Bias in LLM FP4 Pretraining:
Geometric Origin, Systemic Impact, and UFP4 Recipe

论文调研汇报：动机、理论分析、UFP4 配方、实验验证与我的技术判断

FP4 Pretraining E2M1 vs E1M2 / INT4 Shrinkage Bias Random Hadamard Transform Dense & MoE Long-run Training

1. 一句话结论

这篇论文的核心观点是：当前 FP4 训练普遍围绕 E2M1 设计，但在使用 RHT 打散异常值之后， 张量分布会从“动态范围受限”转向“局部分辨率受限”。此时 E2M1 的非均匀量化格点会因为 RTNE bin 几何不对称产生 Shrinkage Bias / 收缩偏差，使信号在多层 GEMM 中系统性衰减。

作者提出 UFP4：使用 E1M2 / INT4-style uniform grid，对 FPROP、DGRAD、WGRAD 三类训练 GEMM 全部使用 RHT，并且只在 dY 上使用随机舍入。 实验显示，UFP4 在 Dense 1.5B、MoE 7.9B、MoE 124B 长训中都比强 E2M1 baseline 更接近 BF16。

2. 论文动机

背景：FP4 是训练降本方向

FP8 已经进入大规模训练系统，FP4 进一步降低显存、带宽和矩阵乘开销。Blackwell / Rubin 级别硬件与 AMD MI350 系列路径都开始强调 FP4。

现状：主流默认 E2M1

MXFP4、NVFP4 等 FP4 训练路径主要围绕 E2M1 数据元素展开。E2M1 有更大的动态范围，因此对原始 outlier-heavy tensor 很有吸引力。

问题：现有方法多在“补救”

现有 recipe 通过 scale、RHT、SR、outlier control、rounding estimator 缓解 FP4 训练不稳定，但大多没有重新审视 E2M1 格点本身是否适合 post-RHT 训练张量。

论文真正想回答的问题不是“RHT 有没有用”，而是： RHT 改变张量分布以后，E2M1 还是不是正确的 FP4 格式？

3. 理论分析：Shrinkage Bias 的几何来源

3.1 Blockwise FP4 quantization

对一个 block B，先用 block 内最大值确定 scale，再将归一化值映射到 FP4 codebook：

s_B = max(|x_j|) / g_max
q_i = ρ_G(x_i / s_B), q_i ∈ G

其中 G 是 E2M1、E1M2 或 INT4 的 normalized codebook；ρ_G 可以是 RTNE 或 SR。 论文关注 RTNE 下非均匀格点产生的系统性误差。

3.2 Shrinkage Bias 定义

令 t = |x| / s 表示归一化幅值。作者定义：

b_G(P) = Et∼P[ ρ_G(t) - t ]

当 b_G(P) < 0 时，量化后的幅值在期望上小于原始幅值，即产生 toward-zero 的系统性收缩。

3.3 RTNE bin 不对称导致负偏差

对某个内部量化 level q_i，其 RTNE bin 左右宽度为：

ℓ_i = (q_i - q_{i-1}) / 2
r_i = (q_{i+1} - q_i) / 2

若 bin 内局部密度近似均匀，条件期望误差为：

E[ρ_G(t)-t | t ∈ B_i] = (ℓ_i - r_i) / 2
= (2q_i - q_{i-1} - q_{i+1}) / 4

因此，只要 r_i > ℓ_i，RTNE 的期望误差就是负数。 这不是普通的随机量化噪声，而是由 codebook 几何结构决定的系统偏差。

格式 非负 magnitude levels 几何性质 结论
E2M1 {0, 0.5, 1, 1.5, 2, 3, 4, 6} 非均匀间距，在 spacing transition 处 bin 左右不对称 会产生 Shrinkage Bias
E1M2 均匀格点风格 相邻 level 间距一致，ℓ_i = r_i 消除该几何来源的偏差
INT4 均匀整数格点 同样是 uniform grid 同样避免该类几何偏差
论文给出的典型例子：E2M1 中 q_i = 2 时，RTNE bin 为 (1.75, 2.5)，ℓ_i = 0.25，r_i = 0.5， 条件期望误差为 -0.125。也就是说，即使局部分布是均匀的，RTNE 也会把幅值系统性往小了舍入。

4. 系统影响：偏差如何在网络里放大

4.1 GEMM 中的 coherent attenuation

对 GEMM Z = ABT，令量化后的 operand 为 Â = Q(A)、B̂ = Q(B)。 作者将量化张量投影回 BF16 原张量方向：

α_A = <Â, A>_F / ||A||²_F
Â = α_A A + R_A, 其中 <R_A, A>_F = 0

若 α_A < 1，说明量化后的 A 在原始 A 方向上的主信号分量被削弱。B 同理。 带入 GEMM 可得：

Z_q = ÂB̂T
= α_A α_B ABT + α_A A R_BT + α_B R_A BT + R_A R_BT

当 residual noise 与主信号不相干时，主信号近似被 η ≈ α_A α_B < 1 衰减。

4.2 多层累积不是零均值噪声，而是乘法衰减

∏k=1..K η_k = ∏k=1..K(1-δ_k) ≈ exp(-Σk=1..K δ_k)

这解释了为什么 shrinkage bias 比普通噪声危险：普通零均值噪声可能部分抵消，但系统性收缩会沿深层路径持续相乘。 特别是 fwd_y 和 bwd_dx 属于 non-leaf outputs，会继续传向后续层或前序层； 而 bwd_dw 是 leaf gradient，主要直接交给 optimizer。

5. RHT 为什么会放大 E2M1 问题

RHT 的本意是用正交旋转分散 outlier energy，提高 codebook bucket utilization。 但论文指出：RHT 会把原来极端尾部的 mass 拉回中等幅值区域，使量化瓶颈从“动态范围”转为“局部分辨率”。

RHT 前

原始训练张量可能有强 outlier。E2M1 的大动态范围可以减少 clipping / saturation，因此 raw tensor 上 E2M1 可能优于 E1M2。

RHT 后

outlier 被打散，分布更集中在中间 bucket。此时局部分辨率更重要，E2M1 的非均匀 bin 反而把 mass 推入最不对称的 RTNE 区间，放大 shrinkage。

因此，论文的关键判断是： Full-RHT 不是天然有害；有害的是 E2M1 与 post-RHT tensor regime 的不匹配。

6. UFP4 方法：Uniform FP4 Recipe

6.1 核心设计原则

当 RHT 已经把训练张量从 dynamic-range-limited regime 转到 local-resolution-limited regime， 4-bit grid 应优先保留局部幅值，而不是继续优先追求极端动态范围。

6.2 UFP4 配方

配置项 强 E2M1 baseline UFP4
Data element format E2M1 E1M2 / INT4-style uniform grid
Quant block size 1×16 1×16
Scale hierarchy FP32 single-level FP32 single-level
RHT scope bwd_dw only fwd_y + bwd_dx + bwd_dw
RHT block size 16 16
SR scope dY dY
2D weight scaling 不使用 不使用

6.3 三个训练 GEMM 的具体流程

BF16 Activation X → RHT along input-channel → Q-to-E1M2 → FPROP FP4 GEMM: Y = XWᵀ
BF16 Gradient dY → RHT + SR → Q-to-E1M2 → DGRAD FP4 GEMM: dX = dY W
dY + X → RHT on reduction dim → Q-to-E1M2 → WGRAD FP4 GEMM: dW = dYᵀ X → Optimizer updates FP32 master weights

RHT 的作用维度是每个 GEMM 的 shared reduction dimension： fwd_y 对 input-channel 维度旋转，bwd_dx 对 output-channel 维度旋转，bwd_dw 对 batch-token 维度旋转。

7. 实验设计与结果

7.1 实验整体设计

作者将实验组织为五个问题：RHT 是否改变最优 4-bit grid；UFP4 是否降低 BF16-relative loss gap； 优势是否跨模型规模保持；哪些 recipe component 有效；RHT 是否能高效 fused 到 FP4 quantization。

所有 end-to-end FP4 对比尽量匹配数据、架构、optimizer、LR schedule、batch、评估频率和训练长度； E2M1 baseline 通过受控配置搜索选择，避免用弱 baseline 放大 UFP4 优势。

8. 技术判断与汇报时可强调的点

最重要的贡献

这篇论文不是简单提出一个新 recipe，而是把 FP4 训练不稳定的一部分原因归因到 format grid geometry：E2M1 的非均匀 RTNE bin 会产生 toward-zero shrinkage。

对 NVFP4 思路的挑战

它并不否定 E2M1 在 raw outlier-heavy tensor 或 inference 中的价值，而是指出： 训练中一旦结合 RHT，uniform grid 可能比更大动态范围更重要。

对硬件的含义

如果未来训练硬件只把 E2M1 当作一等 FP4 primitive，可能限制 FP4 training recipe 的上限。 论文建议 E1M2 / INT4-style uniform grid 也应成为 first-class training primitive。

对算法实现的含义

scale hierarchy、block size、rounding estimator、outlier control 仍然重要； UFP4 是 orthogonal 的 format-level 改动，可与更好的 scale、rounding、QAT 稳定技术组合。

9. 局限与需要谨慎解读的地方

  • 论文仍然显示 FP4 与 BF16 存在 residual gap，UFP4 是降低 loss degradation，不是完全消除差距。
  • 实验主要基于作者训练栈和模型族；外部可复现性取决于是否释放代码、kernel 和完整 recipe 细节。
  • 论文中的主对比使用 FP32 single-level scale、1×16 block、不启用 2D weight scaling，是为了控制变量；这不代表它已经探索了 UFP4 的最优 scale hierarchy。
  • E2M1 仍适合部分 raw outlier-heavy tensor 或 inference 场景；论文主张是“训练硬件应额外支持 uniform FP4”，不是彻底废弃 E2M1。

10. 汇报用主线

第一步：FP4 训练降本很诱人，但主流硬件和 recipe 默认 E2M1。

第二步：作者发现 E2M1 的非均匀 RTNE bin 有几何性收缩偏差，且这种偏差会在 GEMM 链路里乘法累积。

第三步：RHT 虽然打散 outlier，却把张量推入 local-resolution-limited regime，使 E2M1 的 bin 不对称问题更严重。

第四步：UFP4 用 E1M2 / INT4-style uniform grid，允许 FPROP / DGRAD / WGRAD 全路径 RHT，只在 dY 上 SR。

第五步：Dense 1.5B、MoE 7.9B、MoE 124B 长训和消融实验都支持：uniform grid + full RHT 比 E2M1 baseline 更稳定、更接近 BF16。
